User Stories – Multiple Object Detection System

User Story 1: Image Upload Detection

As a user, I want to upload an image to the system so that multiple objects within the image can be automatically detected and identified.

Acceptance Criteria:

* User can upload an image in supported formats (JPG, PNG).
* System processes the image and detects multiple objects.
* Detected objects are displayed with labels and bounding boxes.


User Story 2: Real-Time Camera Detection

As a user, I want to use a live camera feed so that I can detect and track multiple objects in real time.

Acceptance Criteria:

* User can access the camera through the application.
* System performs real-time object detection.
* Bounding boxes and labels update continuously.


User Story 3: Detection Results Display

As a user, I want to view detected object names and confidence scores so that I can understand the accuracy of the predictions.

Acceptance Criteria:

* Each detected object displays a label.
* Confidence scores are shown for all detections.
* Results are easy to read and interpret.


User Story 4: Detection History

As a user, I want the system to store detection results so that I can review previous detections later.

Acceptance Criteria:

* Detection records are saved in the database.
* Users can view past detection results.
* Detection history includes date, time, and detected objects.


User Story 5: Analytics Dashboard

As a user, I want to see statistics about detected objects so that I can analyze object detection trends.

Acceptance Criteria:

* Dashboard displays object counts.
* Charts and reports summarize detection data.
* Statistics can be filtered by date.


User Story 6: System Administrator Monitoring

As an administrator, I want to monitor system performance so that I can ensure reliable object detection services.

Acceptance Criteria:

* System logs detection activities.
* Performance metrics are available.
* Errors and failures are recorded for troubleshooting.
